# anthro-env 项目中文手册

> 极简入口：[[快速上手.md]]

## 0. 项目背景（为什么会有它）

很多开发者都有同一种感受：Claude Code 很强，拿来当命令行编程助手非常好用。

但真实使用里也有“又爱又恨”的地方：

- 官方 token 成本不低
- 一些地区不支持，使用门槛高
- 账号策略和可用性有不确定性

所以，大家常会接入兼容 Anthropic 协议的第三方网关和模型，来保证能用、好用、持续可用。

问题是，环境变量需要频繁切换，手动改来改去非常容易乱，也很影响效率。

`anthro-env` 就是为了解决这个日常痛点：把配置切换做成一个简单的小工具，让你把时间用在写代码，而不是折腾变量。

## 1. 项目简介

`anthro-env` 是一个面向 macOS 的 Anthropic/Claude Code 环境变量配置管理工具。

核心目标：

- 一条命令切换不同配置（profile）
- 默认安全：API Token 存在 macOS Keychain，不写入 `.env` 明文
- 新手可用：交互式初始化、菜单切换、自动诊断

## 2. 适用场景

- 你有多个模型服务入口（官方、代理、第三方网关）
- 你希望快速切换 `ANTHROPIC_*` 变量而不手改 shell 文件
- 你希望降低误操作和密钥泄露风险

## 3. 安装与首次使用

### 3.1 安装（推荐 Homebrew）

```bash
brew install anthro-env/tap/anthro-env
```

### 3.2 初始化

```bash
anthro-env init
source ~/.zshrc
```

初始化会做这些事：

1. 创建配置目录 `~/.config/anthropic` 和 `profiles/`
2. 自动把 hook 写入你的 shell 配置文件（zsh 优先）
3. 引导你创建第一个 profile
4. `ANTHROPIC_MODEL` 可选，不填时使用网关/服务端默认模型
5. 把 `ANTHROPIC_AUTH_TOKEN` 存入 Keychain
6. 激活当前 profile

## 4. 快速上手命令

```bash
anthro-env menu
anthro-env profile add <name>
anthro-env profile use <name>
anthro-env profile ls
anthro-env profile current
anthro-env profile rm <name>
anthro-env doctor
```

说明：

- `anthro-env menu` 中输入为空（直接回车）会默认退出（等价 `0`）
- `profile ls` 中带 `*` 的是当前激活配置

## 5. 配置与存储结构

### 5.1 本地文件

- profiles：`~/.config/anthropic/profiles/*.env`
- 当前配置：`~/.config/anthropic/current`

### 5.2 敏感信息存储

- Token：`macOS Keychain`
- 服务名：`anthro-env`
- 账号命名：`profile:<name>:anthropic_auth_token`

设计原则：profile 文件存非敏感配置（如 `ANTHROPIC_BASE_URL`、模型名），密钥与配置分离。

## 6. 常见工作流

### 6.1 新增配置并切换

```bash
anthro-env profile add my-gateway
anthro-env profile use my-gateway
```

### 6.2 菜单方式切换

```bash
anthro-env menu
```

### 6.3 查看当前配置

```bash
anthro-env profile current
```

### 6.4 环境诊断

```bash
anthro-env doctor
```

`doctor` 会检查：

- 配置目录是否正常
- profile 是否存在
- shell hook 是否安装
- Keychain token 是否可访问

## 7. 配置演进建议

当 profile 数量变多时，建议保持命名一致、定期清理不用配置，并在关键改动后运行 `anthro-env doctor` 做健康检查。

详细可参考：`docs/配置演进与维护建议.md`

## 8. 故障排查

### 8.1 切换后当前终端变量没生效

```bash
source ~/.zshrc
anthro-env doctor
```

通常是 hook 未加载。

### 8.2 Keychain 存取失败

- 确认系统钥匙串处于解锁状态
- 重新执行 `anthro-env profile add <name>` 并录入 token
- 检查是否有企业安全策略限制 `security` 命令

### 8.3 profile 不存在

检查目录：

```bash
ls ~/.config/anthropic/profiles
```

## 9. 开发者说明（贡献）

本地开发：

```bash
make fmt
make test
make build
```

代码结构：

- `cmd/anthro-env`：CLI 入口
- `internal/core`：配置管理、hook、doctor
- `internal/secure`：Keychain 封装
- `internal/ui`：菜单输入与交互逻辑
